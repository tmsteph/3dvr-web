Three levels of the same account

The system starts practical and gets deeper only when the work calls for it.

Portal

Cloud workspace and identity

Keep billing, CRM, notes, messaging, projects, and AI on one account instead of scattering the work across disconnected tools.

  • Start here: subscriptions, contacts, notes, support, and daily operating lanes.
  • Same account: one identity across planning, delivery, and collaboration.
Browser

Runtime, launcher, and media environment

The portal is also becoming a browser-shaped workspace: app launcher, multi-panel tools, media experiments, AI workbench, and lighter desktop-like flows that run anywhere.

  • Run in tabs today: launcher, notes, CRM, media, and AI tools.
  • Grow into a workspace: same apps, but with more context and control.
OS

TommyOS direction for deeper control

Long term, 3dvr is moving toward a Linux-based operating layer for devices, local servers, hardware experiments, and a calmer builder environment that does not depend on closed platforms.

  • Start practical: Pocket Workstation, Debian experiments, and open workflows.
  • Deeper later: same identity, same apps, deeper levels of control.

What this changes

3dvr can stay simple for a new customer while still pointing toward a stronger builder environment.

Start with service

A customer can begin with a website, a support lane, or the portal dashboard without learning a new operating system first.

Keep the path open

The same work can later expand into portable workstations, Debian browser experiments, local servers, and deeper Linux control.
